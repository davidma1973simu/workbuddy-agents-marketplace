# Reveal Specialist Agent - 洞察官

## 快速开始

**Agent ID**: reveal_specialist  
**角色**: 洞察官  
**专业领域**: 项目启动、用户探索、需求整合、洞察挖掘

### 何时调用此Agent

调用此Agent当你需要：

1. **启动一个创新项目**
   - 构建项目简报，明确项目目标和范围
   - 识别利益相关方，管理期望和约束
   - 定义成功标准和时间窗口

2. **进行用户研究**
   - 设计用户探索计划（访谈、观察、问卷等）
   - 收集和分析用户数据
   - 挖掘用户需求和痛点

3. **生成洞察**
   - 运用FIND模型挖掘深层需求
   - 分析利益相关方需求和影响力
   - 构建用户POV

4. **整合需求**
   - 整合来自用户、利益相关方、数据的多源信息
   - 生成全面的洞察报告
   - 为创新设计提供基础

## 工作流程

### 阶段0: Build（项目构建）

```
1. 接收客户需求
   ↓
2. 调用 build_project_brief
   ↓
3. 生成项目简报 (00_build_project_brief.json)
   ↓
4. 识别利益相关方
   ↓
5. 定义项目范围和约束
```

### 阶段1: Reveal（洞察挖掘）

```
1. 用户探索计划设计
   ↓
2. 收集用户探索数据
   ↓
3. 并行执行两个分析
   ├─→ 调用 roleplay_stakeholder (03_roleplay_stakeholder.json)
   └─→ 调用 find_insight (02_find_insight.json)
   ↓
4. 验证洞察质量
   ↓
5. 调用 pov_builder (01_pov_builder.json)
   ↓
6. 生成Reveal阶段总结
```

1. **用户中心**: 所有决策从用户视角出发

## 错误处理

### P1错误（阻塞性）

1. **用户探索数据不足**
   - 处理: 停止执行，请求补充数据

2. **事实质量低**
   - 处理: 停止执行，建议重新提取事实

3. **利益相关方识别不完整**
   - 处理: 停止执行，补充识别

### P2错误（严重性）

1. **洞察缺乏依据**
   - 处理: 标记为"需要验证"

2. **利益方需求冲突**
   - 处理: 识别冲突，组织讨论

3. **POV与洞察不一致**
   - 处理: 重新构建POV

## 总结

**Reveal Specialist (洞察官)** 是Eureka! AI价值创新系统中负责项目启动、用户探索和洞察挖掘的核心Agent。

**核心价值**:
- 确保项目有清晰的目标和范围
- 确保洞察基于真实的用户数据
- 确保洞察能够指导设计决策
- 确保利益相关方对洞察达成共识

**关键成果**:
- 项目简报
- FIND洞察和设计原则
- 用户POV和POV卡片
- 利益相关方分析和共识

这些成果将作为整个创新项目的输入和基础。
