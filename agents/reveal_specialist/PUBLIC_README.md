专业的洞察挖掘专家，帮助你启动创新项目、进行用户研究、生成深度洞察，为价值创新奠定坚实基础。

## 📋 使用提示

### 何时调用？

✅ **调用此Agent当你需要：**

1. **启动创新项目**
   - "我有一个创新想法，需要启动项目"
   - "帮我构建项目简报，明确目标和范围"

2. **进行用户研究**
   - "需要设计用户访谈或观察计划"
   - "帮我分析用户数据，挖掘需求"

3. **生成洞察**
   - "从用户数据中挖掘深层需求"
   - "构建用户POV（问题-机会-愿景）"

4. **整合需求**
   - "整合用户、利益相关方的需求"
   - "为创新设计提供洞察基础"

### 如何使用？

**简单调用**：
```
"启动Reveal Specialist，帮我处理以下需求：[你的项目描述]"
```

**分步调用**：
```
第一步："使用build_project_brief构建项目简报"
第二步："使用roleplay_stakeholder分析利益相关方"
第三步："使用find_insight挖掘FIND洞察"
第四步："使用pov_builder构建用户POV"
```

### 核心能力

- 🔍 **用户研究**: 访谈、观察、数据分析
- 💡 **洞察挖掘**: FIND模型（事实→解读→需求→洞察）
- 🤝 **利益相关方**: Buy Features方法，达成共识
- 📄 **需求整合**: 生成用户POV，指导创新设计

## 🎯 快速开始示例

### 示例1: 完整项目启动

```
用户: "我需要解决鞋服零售门店的用户选择困难症，启动一个创新项目"

Reveal Specialist将：
✅ 构建项目简报（背景、挑战、目标、用户画像）
✅ 识别利益相关方（6个关键角色）
✅ 设计用户探索计划（访谈、观察）
✅ 挖掘FIND洞察（10个事实→5个解读→4层次需求→核心洞察）
✅ 生成用户POV（"Lisa在选择时面对200+款商品，无法决策，因为她需要更聪明的选择而非更多选项"）
✅ 输出6个完整文件（JSON + 可视化）
```

### 示例2: 生成洞察

```
用户: "我有用户访谈数据，帮我挖掘洞察"

Reveal Specialist将：
✅ 分析用户探索数据
✅ 提取10个关键事实
✅ 生成5个行为解读
✅ 识别4层次需求（显性、隐性、情感、深层）
✅ 生成核心洞察陈述
✅ 提供设计原则（减少、引导、赋能、个性化、透明）
```

## 📁 输出内容

使用此Agent，你将获得：

1. **项目简报** (00_build_project_brief.json)
   - 背景、核心挑战、SMART目标
   - 用户画像、时间窗口、利益相关方

2. **FIND洞察** (02_find_insight.json)
   - 事实、解读、需求、洞察
   - 设计原则和设计建议

3. **用户POV** (01_pov_builder.json)
   - POV陈述和POV卡片
   - 成功标准和风险识别

4. **利益相关方分析** (03_roleplay_stakeholder.json)
   - Buy Features分析（总分12分约束）
   - 影响力评估（1-3分）
   - 需求优先列表

5. **可视化界面**
   - POV卡片 (pov_card.html)
   - 需求优先列表 (stakeholder_requirements_list.txt)
