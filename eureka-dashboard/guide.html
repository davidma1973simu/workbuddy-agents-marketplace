🎯 产品定位

Eureka 是什么？

Eureka 是一个创新项目成果仪表板，帮助团队把设计思维四阶段（Reveal → Inspire → Shape → Exam）的所有过程产出，沉淀成一个可视化、可分享、可导出的在线项目库。

📁
项目库
多项目集中管理，状态一目了然，支持搜索/筛选/归档。
✏️
四阶段编辑器
分阶段录入产出——画像、HMW、概念方案、测试报告，结构清晰。
📊
可视化看板
甘特图、MAP 雷达、AHA 雷达、商业模式画布，自动渲染。
📤
一键导出
PDF 报告 / Markdown 文档 / PPT 幻灯片，三种格式随时交付。
✦
纯前端，零部署负担
所有数据存储在浏览器 localStorage，无需注册账号、无需服务器。打开网页即用，刷新不丢数据。
📦 准备

开始前，你需要准备什么？

不需要装任何软件。打开浏览器，30 秒搞定。

🌐
现代浏览器
Chrome / Edge / Safari / Firefox 均可。推荐 Chrome，体验最佳。
🔗
打开仪表板
→ 在线访问 Eureka Dashboard
📝
项目原始资料
把设计思维过程中的便利贴、文档或照片整理好，录入更高效。
👥
团队分工（可选）
每阶段可由不同成员负责录入，最终汇入同一个项目即可。
⚠️
同一台电脑 = 同一份数据
数据存在浏览器本地，换设备或清理浏览器缓存会导致数据丢失。重要项目记得定时「导出 Markdown」备份。
🔄 全流程

完整使用流程

从建项目到交付成果，共 6 步，平均全程 2–4 小时（取决于内容丰富度）。

🆕
Step 01
新建项目
点击右上角「＋ 新建项目」，填写项目名称、简报描述、目标用户、适用场景，选择初始状态（草稿/进行中）。
系统自动生成唯一 ID，立即出现在左侧项目列表。
项目名称 目标用户 场景描述
🔍
Step 02
录入 Reveal 阶段
在编辑器「Reveal」Tab 中，录入项目简报五要素（背景/机会/假设/范围/团队）、POV 洞察三段式、用户画像（最多 4 个）、利益相关者、场景地图触点；同时可粘贴用户访谈记录原文和亲和图聚类分析。
POV 用户画像 利益相关者 HMW 访谈记录 亲和图
💡
Step 03
录入 Inspire 阶段
在「Inspire」Tab 中添加创意方案（可打分：可行性/价值性/创新性），标记最佳创意并填写选择理由；还可录入 Crazy 8 草图——用文字描述 8 个快速发散方向，留存头脑风暴过程。系统自动生成评估矩阵和高亮最佳方案。
创意列表 评分矩阵 Crazy 8 最佳创意
🎨
Step 04
录入 Shape 阶段
在「Shape」Tab 中填写概念名称 / 全局描述 / 功能特性（最多 6 个）、六幕体验故事，以及 MAP 商业价值雷达分值（市场潜力/用户增长/优势壁垒，1–10）。
概念方案 功能特性 体验故事 MAP 雷达
✅
Step 05
录入 Exam 阶段
在「Exam」Tab 中录入 AHA 价值雷达（顿悟/高光/进步）、电梯呈现五段式、30-60-90 天迭代计划，以及商业模式画布九格内容。
AHA 雷达 电梯呈现 迭代计划 商业画布
🗂️
Step 05.5
查看专家过程产出物
右侧最底部「🗂️ 专家过程产出物」面板，按四阶段列出全部 15 项产出物。每项卡片独占一行：名称靠左、状态徽章（✓ 已填写 / 待填写 / BETA）靠右；第二行显示产出物类型标签；第三行为内容摘要（2 行截断）；第四行操作按钮：⬇ 下载 MD、✨ AI 优化（Prompt 模板）、✏ 编辑（直跳对应 Tab）。完成度进度条显示在标题右侧（如 11 / 15 已完成）。
15 项产出物 状态感知 下载 MD AI Prompt
🚀
Step 06
导出 & 交付
在右侧「可交付资产」区域，根据项目进度解锁：PDF 报告（完整打印）、Markdown（复制到剪贴板）、PPT 幻灯片（下载 HTML 格式）。左上角「📅 项目计划」Tab 可追踪四阶段实际进度和时间线。

FEATURE SPOTLIGHT
🗂️ 专家过程产出物面板
共 15 项产出物

每个项目仪表板底部有一个「专家过程产出物」面板，按四阶段（Reveal / Inspire / Shape / Exam）分四列展示全部 15 项创新过程产出。该面板实时感知数据状态，是项目过程留存和 AI 协作的核心入口。

📐 卡片排版结构
第 1 行：图标 + 名称（超长省略）+ 状态徽章
第 2 行：产出物类型标签（彩色）
第 3 行：内容摘要（最多 2 行截断）
第 4 行：操作按钮（下载 / AI / 编辑）
✓ 三种状态
✓ 已填写 左侧绿色竖线 + 绿色边框
待填写 灰色边框，显示「去填写」按钮
BETA 虚线边框，即将支持
⚡ 三大操作
⬇ 下载：将该产出物导出为 Markdown 文件
✨ AI 优化：弹出专属 Prompt 模板，复制给 AI
✏ 编辑：直接跳转到对应编辑器 Tab
💡 小贴士：每次在编辑器中保存数据后，该面板会自动更新状态。右上角的进度数字（如 11 / 15 已完成）可帮助你追踪整个项目过程的完整度。

💡 避坑指南

这些坑不用踩

基于真实使用场景整理，5 分钟省去 5 小时迷糊。

🗑️
删除 ≠ 永久消失
从项目列表删除会先进入回收站（左侧 Tab 切换），可恢复。彻底删除才是真的消失。误删不要慌，先看回收站。
💾
每次修改都要点「保存项目」
编辑器不会自动保存。每个 Tab 修改后，记得点底部「保存项目」。关闭浏览器前务必保存一次。
📱
手机可以查看，不建议录入
仪表板支持响应式，手机可查看已有项目成果，但大量文字录入建议在桌面端完成，体验更流畅。
📤
重要项目导出 Markdown 备份
清理浏览器数据会导致 localStorage 清空。建议每次重要更新后「导出 Markdown」，存到云盘作为版本备份。
🎯
PPT 导出是 HTML 文件
导出的 PPT 是一个 HTML 幻灯片，用浏览器打开后按 F11 全屏即可演示。如需转成真正的 .pptx，可截图后手工整理或告知辅导老师协助。

🔍 Reveal Specialist
洞察专家
负责用户洞察阶段，从 Project Brief 生成 POV、FIND 洞察和利益相关方分析，帮你看清「用户真正需要什么」。
👁
pov_builder · POV 构建师
将项目简报转化为「用户画像 + 场景触点 + 痛点 + POV 陈述」四维结构。支持 AI 自动生成或 GUI 交互式手动填写。
🔬
find_insight · FIND 洞察器
运用 FIND 模型（Facts→Interpret→Need→Design insight）从用户旅程事实中挖掘深层需求，生成精炼洞察卡片。
🎭
roleplay_stakeholder · 利益相关方模拟
AI 扮演多个利益相关方角色，模拟各方需求、影响力和潜在冲突，输出关系地图。
💬 唤醒示例
「请用 pov_builder 帮我构建针对 20-35 岁都市白领女性购物场景的用户 POV」
💡 Inspire Specialist
启发专家
负责创意生成阶段，从 POV 出发产出 HMW 机遇、交叉激发创意，并提炼最佳方案，帮你打开「可能性空间」。
❓
HMW_Reframer · HMW 机遇重构器
从 4 个维度重构用户 POV，生成 6-8 个「How Might We...」创新机遇，并按用户价值 × 商业可行双维度筛选最佳 HMW。
⚡
NCO_Crosslinker · 新酷界外交叉激发器
强制从「新（New）/ 酷（Cool）/ 界外（Out of box）」三类生成灵感源，与最佳 HMW 交叉激发 20-30 个全新创意。
💎
Idea_Distiller · 创意提炼师
对创意列表进行评分（可行性/价值性/创新性）、聚类、提炼，选出最有潜力的核心创意并给出「为什么是它」的理由。
💬 唤醒示例
「用 NCO_Crosslinker 对『如何让购物决策更轻松』这个 HMW 进行交叉激发，生成 20 个创意」
🎨 Shape Specialist
构建专家
负责方案构建阶段，对最佳创意进行四维压力测试，构建完整概念方案，并写出用户视角六幕体验故事。
🔨
Solution_Interrogator · 方案拷问员
从「用户 / 技术 / 商业 / 利益方」四个维度对最佳创意进行压力测试，发现风险与盲点，迫使方案自我完善。
🏗
Solution_Build · 方案构建师
整合压力测试结果，构建完整概念方案：概念名称 + 核心功能特性（最多 6 个）+ AHA 价值设计，输出可直接填入仪表板的结构化内容。
🎬
Storyboard_Scriptor · 故事脚本师
以用户视角撰写六幕体验故事脚本：用户挑战→POV洞见→接触→使用→分享结果→持续感受，每幕含场景、情感和 AHA 瞬间。
💬 唤醒示例
「用 Solution_Interrogator 从四个维度拷问『AI 智能购物推荐』这个创意方案」
✅ Exam Specialist
验证专家
负责验证评估阶段，量化方案的 AHA 和 MAP 价值，撰写电梯演讲，制定迭代计划，完成商业模式画布。
📊
AHA_MAP_Evaluator · 价值评估师
汇总测试报告，量化评估 AHA 用户价值（顿悟/高光/进步）和 MAP 商业价值（市场潜力/用户增长/优势壁垒），生成双雷达图数据。
🎤
Pitch_Composer · 电梯呈现撰写师
生成 60 秒电梯演讲脚本（五段式：问题/方案/用户/价值/行动），语言精炼有力，适合向投资人或决策者呈现。
📅
Project_Iteration · 迭代计划师
制定 30-60-90 天分阶段迭代计划，每个节点包含具体里程碑、成功指标和资源需求。
🖼
BusinessModel_Painter · 商业模式画家
完整填写商业模式画布 9 格（价值主张/客户细分/渠道通路/客户关系/收入来源/核心资源/关键业务/重要合作/成本结构）。
💬 唤醒示例
「用 Pitch_Composer 为『智能购物推荐系统』写一段 60 秒电梯演讲」

🔌 API 配置

如何配置 AI 与调用 Sub-Agent

Eureka 支持两种 AI 调用方式：Dashboard 内直接配置 API Key，或通过 WorkBuddy 调用已集成的 Skills。选择适合你的方式开始使用。

METHOD 01
🔑 在 Dashboard 内配置 API Key
步骤 1：打开 AI 设置
进入 Eureka Dashboard，点击页面右上角「⚙️ AI 设置」按钮，弹出配置面板。
步骤 2：选择服务商
支持 OpenAI、DeepSeek、通义千问，或选择「自定义」输入其他兼容 OpenAI 接口格式的服务商。
步骤 3：填写配置
输入 API Key、模型名称（如 gpt-4、deepseek-chat）、Base URL（如使用自定义服务商）。
步骤 4：测试并保存
点击「测试连接」验证 API 可用性，确认无误后点击「保存配置」。配置将存储在浏览器本地。
⚠️ 安全提示：API Key 仅存储在浏览器 localStorage 中，不会上传到任何服务器。建议个人使用，不要在公共电脑上保存敏感 Key。团队共享场景建议使用 WorkBuddy 集成方式。
METHOD 02 · RECOMMENDED

如果你已经在使用 WorkBuddy，可以直接调用 Eureka 系列 Skills，无需在 Dashboard 中重复配置 API Key。WorkBuddy 会自动管理 API 调用和上下文传递。

🔧 常见问题排查

❌ "未配置 AI 接口" 提示
表示当前没有可用的 API 配置。点击 Dashboard 右上角「⚙️ AI 设置」进行配置，或切换到 WorkBuddy 环境使用。
❌ "API 连接失败" 错误
检查 API Key 是否正确、是否已过期、是否有足够的余额。自定义 Base URL 时需确保接口兼容 OpenAI 格式。
❌ 响应缓慢或超时
可能是网络问题或模型服务商响应慢。尝试切换模型（如从 gpt-4 切换到 gpt-3.5-turbo）或检查网络连接。
❓ WorkBuddy 中找不到 Eureka Skills
确保 WorkBuddy 已更新到最新版本。Skills 列表中搜索 "Eureka" 或 "创新" 即可找到相关技能包。
❓ 常见问题

遇到问题先看这里

刷新页面，数据还在吗？ ⌄
在的。数据存储在浏览器 localStorage，普通刷新不影响。但清除浏览器数据/缓存或换浏览器会导致数据消失。重要项目请定期导出 Markdown 备份。
多人协作可以用同一个项目吗？ ⌄
目前不支持实时多人协同（数据在各自浏览器本地）。推荐一人主导录入，其他成员提供素材；或导出 Markdown 后分工编辑，再由一人汇总回 Eureka。
导出的 PDF 版面乱了怎么办？ ⌄
PDF 使用浏览器打印生成。打印设置中调整纸张方向（横向效果更好）、缩放比例（推荐 80%），关闭「页眉页脚」选项。Chrome 效果优于其他浏览器。
雷达图/甘特图不显示？ ⌄
雷达图需要填写对应评分数据才会渲染。甘特图需要在「项目计划」Tab 中设置至少一个阶段的日期。填完保存后刷新页面或重新点击该项目即可。
可以把项目迁移到另一台电脑吗？ ⌄
可以。在当前电脑导出 Markdown，在新电脑「上传创新文档」（导航栏按钮）导入内容，手工核对后重新保存即可。未来版本将支持 JSON 完整导入导出。
项目进度是怎么自动计算的？ ⌄
系统根据「项目计划」Tab 中各阶段的状态（completed / in_progress / pending）自动推算整体 progress 百分比。状态设置越完整，进度越准确。也可以在基本信息 Tab 手动设置进度。
